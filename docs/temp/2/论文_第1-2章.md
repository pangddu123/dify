# 基于Java Web的萌宠乐园在线管理平台的设计与实现

---

# 摘要

## 中文摘要

近年来，随着中国社会经济的持续发展和居民生活水平的不断提高，宠物已逐渐成为越来越多家庭的重要成员。据相关行业报告统计，2023年中国宠物行业市场规模接近3000亿元，养宠家庭数量超过1亿户，宠物经济已成为消费市场中不可忽视的组成部分。然而，当前大量中小型宠物乐园和宠物服务机构的信息化管理水平仍然较低：宠物信息管理依赖纸质记录或Excel表格，服务预约主要通过电话完成，商品销售渠道单一，客户互动平台缺失，这些问题严重制约了宠物乐园的运营效率和服务质量。

针对上述问题，本文设计并实现了一个基于Java Web的萌宠乐园在线管理平台。系统采用前后端分离架构，后端基于Spring Boot 2.7框架构建，结合MyBatis-Plus 3.5.3持久层框架实现对MySQL数据库的高效访问，并集成JWT（JSON Web Token）机制实现用户身份认证与权限管理；前端采用Vue.js 2.6框架，配合Vue Router实现路由管理、Vuex实现集中式状态管理，并使用Element UI 2.15组件库构建用户界面，通过Axios库完成与后端RESTful API的数据交互。

平台包含用户管理、宠物信息管理、服务预约、商品与订单、社区论坛和后台管理六大核心功能模块，分为前台用户界面与后台管理界面两部分，支持普通用户和管理员两种角色的差异化权限控制。经过系统功能测试与验证，本平台运行稳定、功能完善，能够有效提升宠物乐园的管理效率和服务质量，为中小型宠物乐园提供了一套低成本、易部署的信息化解决方案。

**关键词：** 萌宠乐园；在线管理平台；Spring Boot；Vue.js；前后端分离

## Abstract

With the continuous development of China's social economy and the steady improvement of residents' living standards, pets have gradually become important members of an increasing number of households, and the pet industry has ushered in unprecedented development opportunities. According to industry reports, China's pet industry market scale approached 300 billion yuan in 2023, with more than 100 million pet-owning households. Yet the informatization level of many small and medium-sized pet kingdoms remains low: pet information management relies on paper records or spreadsheets, service appointments are mainly handled via telephone, merchandise sales channels are limited, and customer interaction platforms are largely absent. These problems restrict operational efficiency and service quality.

To address these issues, this paper designs and implements an online management platform for pet kingdoms based on Java Web technology. The system adopts a front-end and back-end separation architecture. The back-end is built on Spring Boot 2.7, combined with the MyBatis-Plus 3.5.3 persistence framework for efficient MySQL access, and integrates the JWT mechanism for authentication and permission control. The front-end uses Vue.js 2.6, with Vue Router for routing, Vuex for centralized state management, and the Element UI 2.15 component library for UI construction. Data exchange between layers is performed through Axios calling back-end RESTful APIs.

The platform comprises six core modules—user management, pet information management, service appointment, merchandise and orders, community forum, and back-end administration—and is divided into a front-end user interface and a back-end management interface with differentiated permissions for regular users and administrators. System testing and verification show that the platform runs stably with comprehensive functionality, effectively improving the management efficiency and service quality of pet kingdoms, and providing a low-cost, easily deployable informatization solution for small and medium-sized pet kingdoms.

**Keywords:** Pet Kingdom; Online Management Platform; Spring Boot; Vue.js; Front-end and Back-end Separation

# 第1章 绪论

## 1.1 研究背景与意义

近年来，中国宠物行业呈现出蓬勃发展的态势，市场规模持续快速扩张。根据《2023—2024年中国宠物行业白皮书》的数据，2023年中国城镇宠物（犬猫）消费市场规模已达2793亿元，较上一年度增长显著，养宠家庭数量突破1亿户。随着城市化进程加速、人口结构变化以及居民消费观念转变，宠物经济已从最初的宠物食品、用品等基础消费，逐步拓展至医疗、美容、寄养、训练、保险、殡葬等多个细分领域，形成了较为完整的产业链。

在产业链不断完善的背景下，线下宠物乐园作为集展示、互动、服务与社交于一体的综合性场所，受到了广大宠物爱好者的青睐。然而，当前大量中小型宠物乐园在日常运营管理中仍面临明显的信息化不足：一是宠物信息管理多依赖纸质档案或Excel表格，效率低下且易出现数据丢失和更新不及时等问题；二是服务预约主要通过电话或微信私聊完成，缺乏统一的在线预约渠道，预约信息混乱、时间冲突频发；三是商品销售渠道单一，主要依赖线下门店，难以满足消费者日益增长的线上购物需求；四是客户互动平台缺失，宠物主人之间难以进行经验分享和交流，客户黏性较低。

信息化管理是解决上述问题的有效途径。通过构建功能完善的在线管理平台，可以实现宠物信息的数字化管理、服务预约的在线化处理、商品销售的多渠道拓展以及客户互动的社区化运营，从而提升运营效率、优化客户体验，并为管理者提供数据驱动的决策支持。

基于上述背景，本文设计并实现了一个基于Java Web的萌宠乐园在线管理平台，旨在为中小型宠物乐园提供一套低成本、易部署、功能聚焦的信息化解决方案。平台整合了宠物信息展示与管理、在线服务预约、商品销售与订单管理、社区论坛交流等核心功能，采用当前主流的Spring Boot和Vue.js技术栈、前后端分离架构进行开发，具有良好的可扩展性和可维护性。本系统的研究与实现不仅具有实际应用价值，也可为同类型中小型服务平台的信息化建设提供参考。

## 1.2 国内外研究现状

### 1.2.1 国外宠物管理平台发展现状

国外宠物管理平台起步较早，已形成一批用户规模较大的平台型企业。Petfinder成立于1996年，是全球最大的宠物领养信息平台之一，汇聚了北美地区数万家收容所和救助机构的宠物信息，其技术架构支持大规模数据的实时检索和智能推荐。Rover成立于2011年，专注于宠物寄养和遛狗服务，连接宠物主人与经过审核的看护人员，具备完善的预约管理、在线支付、评价反馈和服务跟踪等功能。Chewy成立于2011年，是北美领先的宠物电商平台，其技术平台支撑庞大的商品管理、订单处理和物流配送体系，并提供自动补货和个性化推荐能力。VCA Animal Hospitals为大型宠物医疗连锁机构，其管理平台实现了健康档案、在线挂号、诊疗记录和医疗数据分析等功能，代表了宠物医疗信息化的较高水平。

上述国外平台在功能完善度和技术实现方面均达到较高水平，但主要面向海外市场，其业务模式、服务流程与用户习惯与国内市场存在较大差异，难以直接应用于国内宠物乐园管理场景。

### 1.2.2 国内宠物平台发展现状

随着国内宠物市场的快速发展，也涌现出了一批代表性平台。波奇网成立于2008年，是国内较早进入该领域的综合性平台，业务涵盖宠物电商和宠物社区，但主要聚焦电商领域，缺乏面向宠物乐园的管理工具。宠物之家以宠物社区为核心，提供养宠知识、宠物日记和论坛交流等功能，但功能较为单一，不具备服务管理和商品销售等商业化能力。萌爪医生专注于宠物在线问诊，连接宠物主人与执业兽医提供咨询、健康管理和用药指导服务，填补了宠物医疗信息化领域的部分空白。

然而，上述国内平台多为大型综合平台或垂直领域平台，主要面向C端消费者，缺乏面向中小型宠物乐园和服务机构的专用管理工具。对中小型乐园而言，这些平台功能过于复杂，且缺乏服务预约管理、宠物信息管理和客户关系管理等核心能力，难以满足其实际运营需求。

### 1.2.3 现有系统不足与本系统的差异化

综合分析国内外现有平台的发展现状，可以发现以下共同不足：第一，功能过于复杂，大型综合平台模块众多，部署和维护成本较高，不适合中小型宠物乐园；第二，普遍缺乏针对宠物服务场景的预约管理模块；第三，社区互动形式单一，难以有效提升用户黏性；第四，传统管理软件的采购和服务器成本对中小型商户负担较重。

针对上述不足，本系统在设计上突出以下差异化特色：一是轻量级设计，功能聚焦于宠物乐园的核心业务需求，降低复杂度和部署成本；二是功能聚焦，围绕宠物信息管理、服务预约、商品销售和社区交流四大核心业务展开；三是采用前后端分离架构，前后端可独立开发与部署，便于功能扩展和技术升级；四是强化社区功能，通过论坛模块增强用户间的互动交流，提升平台活跃度。

# 第2章 相关技术介绍

本章介绍萌宠乐园在线管理平台所采用的关键技术，为后续章节的系统设计与实现奠定技术基础。

## 2.1 Spring Boot框架

Spring Boot是由Pivotal团队于2014年推出的Spring快速开发框架，秉承"约定优于配置"的设计理念，在Spring框架的基础上提供自动化配置与开箱即用的功能组件，使开发者能以极少的配置快速构建生产级Spring应用。

Spring Boot的核心特性包括：（1）**自动配置**，通过@SpringBootApplication注解根据项目依赖自动推断并装配Spring组件，例如引入spring-boot-starter-web后即自动配置嵌入式Servlet容器、Spring MVC及HTTP消息转换器；（2）**起步依赖**，将常用依赖按功能分组为一系列spring-boot-starter-*，解决依赖版本冲突和管理繁琐的问题；（3）**内嵌Servlet容器**，内置Tomcat、Jetty或Undertow，应用可以JAR包方式独立运行，简化部署；（4）**Actuator监控**，提供健康检查、性能指标采集和配置查看等运维能力。

相较于传统SSM（Spring + SpringMVC + MyBatis）框架，Spring Boot在配置、依赖管理和部署三方面均具优势：无需编写大量XML配置，依赖版本由Starter统一管理，可直接以JAR包独立运行。

本项目后端采用Spring Boot 2.7.18版本，使用spring-boot-starter-web构建RESTful API服务，使用spring-boot-starter-validation完成请求参数校验，并借助自动配置机制快速搭建完整的后端服务框架。

## 2.2 Vue.js前端框架

Vue.js是由尤雨溪于2014年发布的渐进式JavaScript前端框架，以简洁的API设计、优秀的性能表现和完善的中文文档，迅速成为全球最受欢迎的前端框架之一。

Vue.js的核心特性包括：（1）**响应式数据绑定**，Vue.js 2.x通过Object.defineProperty对数据属性进行劫持，数据变化时自动更新视图，开发者无需手动操作DOM；（2）**组件化开发**，通过.vue单文件组件将模板、逻辑和样式封装为可复用单元，实现代码模块化；（3）**虚拟DOM**，在内存中维护轻量级DOM副本，借助Diff算法仅更新发生变化的部分，提升渲染性能；（4）**指令系统**，提供v-if、v-for、v-model、v-on、v-bind等声明式DOM操作方式。

Vue.js拥有完善的官方生态：Vue Router提供SPA前端路由管理，支持路由守卫与懒加载；Vuex以单一状态树实现应用状态的集中管理与可预测变更；Vue CLI提供脚手架、开发服务器、构建打包等工具链支持。

与React（JSX、函数式、灵活性高但学习曲线较陡）和Angular（大而全、重量级）相比，Vue.js介于两者之间，兼顾轻量灵活与官方生态完整性，学习曲线平缓，特别适合中小型项目快速开发。

本项目前端采用Vue.js 2.6.14版本，结合Vue Router 3.5.3实现路由管理，Vuex 3.6.2管理登录态和购物车等全局状态，并使用Vue CLI 5.0作为项目脚手架。

## 2.3 MyBatis-Plus持久层框架

MyBatis-Plus（简称MP）是苞米豆（Baomidou）团队开发的MyBatis增强工具，秉承"只做增强不做改变"的设计理念，引入后不会影响现有MyBatis功能，同时带来大量开箱即用的增强能力。

MyBatis-Plus的核心特性包括：（1）**通用CRUD接口**，Mapper接口继承BaseMapper<T>即可获得insert、deleteById、updateById、selectById/selectList等常用方法，无需编写XML；（2）**条件构造器**，LambdaQueryWrapper支持以Lambda表达式引用实体属性构建查询条件，避免硬编码字段名，并支持链式调用；（3）**分页插件**，通过PaginationInnerInterceptor拦截器实现自动物理分页，支持MySQL等多种数据库方言；（4）**自动填充**，通过MetaObjectHandler在插入和更新时自动填充创建时间、更新时间等公共字段；（5）**逻辑删除**，通过@TableLogic注解将删除转换为状态更新，保护业务数据完整性。

相较于原生MyBatis逐一编写Mapper和XML，MyBatis-Plus通过通用接口自动生成常用操作，官方数据显示在常规业务中可减少60%以上的代码量，同时完全兼容原生MyBatis，对复杂查询仍可自定义SQL。

本项目持久层采用MyBatis-Plus 3.5.3.1版本，所有Mapper继承自BaseMapper<T>，通过LambdaQueryWrapper构建动态查询条件，并配置了分页插件实现列表数据分页。

## 2.4 MySQL数据库

MySQL是应用最为广泛的开源关系型数据库之一，以高性能、高可靠性、易用性和开源免费的特点，广泛应用于Web应用和企业信息系统。

MySQL的主要特点包括：查询性能优异、具备完备的备份与恢复机制、InnoDB存储引擎支持ACID事务、提供多种存储引擎可按场景选择。InnoDB作为MySQL默认存储引擎，支持行级锁、外键约束以及基于重做日志（Redo Log）和回滚日志（Undo Log）的崩溃恢复，是本项目所采用的引擎。

在数据库设计上，本项目遵循第三范式（3NF），消除数据冗余与传递依赖；针对高频查询字段建立合理的索引，兼顾查询性能与写入开销。字符集采用utf8mb4以完整支持包括Emoji在内的Unicode字符，满足社区论坛等富表情输入场景。数据库负责存储全部业务数据，包括用户、宠物、商品、订单、预约和论坛帖子等。

## 2.5 JWT认证机制

JWT（JSON Web Token）是一种基于JSON的开放标准（RFC 7519），用于在各方之间以紧凑且安全的方式传输信息。凭借无状态、跨域友好和易于扩展等特点，JWT已成为前后端分离架构下广泛采用的认证方案。

JWT由Header、Payload、Signature三部分组成，各部分间以"."分隔。Header声明令牌类型和签名算法；Payload存储用户声明信息（Claims），如用户ID、用户名、角色及过期时间等；Signature对前两部分签名，确保令牌不可篡改。

[图2-1 JWT令牌结构与认证流程示意图]

与传统Session相比，JWT具有四方面优势：（1）**无状态**，用户信息直接编码在令牌中，服务端无需维护会话，天然支持分布式部署；（2）**跨域友好**，通过Authorization请求头传递，不受Cookie同源策略限制；（3）**适配移动端**，无需依赖Cookie机制；（4）**可扩展性强**，自定义声明可携带业务信息，减少数据库查询。

本项目JWT认证流程如下：用户登录时，后端验证用户名和密码，验证通过后使用java-jwt 4.4.0库以HMAC256算法生成有效期24小时的令牌，返回前端并存储于localStorage。此后前端每次请求在Authorization头中以"Bearer Token"形式携带令牌；后端通过拦截器提取并验证令牌，校验通过则将用户信息注入请求上下文，否则返回401要求重新登录。

## 2.6 Element UI组件库

Element UI是饿了么前端团队开源的一套基于Vue.js 2.x的桌面端组件库，提供丰富且设计规范的UI组件，帮助开发者快速构建风格统一的Web应用界面。

Element UI提供的组件覆盖布局（Container、Layout）、导航（NavMenu、Breadcrumb、Tabs）、表单（Form、Input、Select、DatePicker、Upload等）、数据展示（Table、Pagination、Tag、Badge）和反馈（Dialog、Message、Notification、Loading）等常见场景，并内置表单验证能力。其显著特点是设计规范统一、组件数量丰富（60余个）、文档详尽（中英文齐备），并支持通过在线主题编辑器或SCSS变量进行主题定制。

本项目采用Element UI 2.15.14版本构建前台用户界面和后台管理界面的所有页面，包括登录注册、宠物列表、服务预约、商品展示、购物车、社区论坛、后台数据管理等，充分利用其组件能力实现美观统一的交互体验。

## 2.7 RESTful API与前后端分离架构

REST（Representational State Transfer）是Roy Fielding于2000年在其博士论文中提出的软件架构风格，基于该风格设计的Web接口称为RESTful API，已成为Web应用接口设计的主流范式。

RESTful API的核心设计原则包括：（1）**资源导向**，系统实体抽象为资源并通过URI标识，如/users表示用户集合、/users/1表示具体用户；（2）**HTTP方法语义明确**，GET查询、POST创建、PUT更新、DELETE删除；（3）**无状态通信**，每次请求包含处理所需的全部信息，便于服务端水平扩展；（4）**统一接口**，通过统一的资源标识、操作语义和表现形式（如JSON）简化整体架构。

[图2-2 前后端分离架构示意图]

状态码的合理使用对接口语义至关重要。本项目遵循以下规范：200请求成功、201资源创建成功、400请求参数错误、401未认证、403无权限、404资源不存在、500服务器内部错误。

前后端分离架构下，前端以SPA形式独立运行，通过AJAX调用后端RESTful API并以JSON交换数据。该架构具有如下优势：前后端可独立开发、测试和部署；通过标准化API降低系统耦合度；SPA带来无刷新页面交互，提升用户体验；后端API可被Web、移动端、小程序等多端复用。

本项目前端Vue CLI开发服务器通过proxy代理解决与后端Spring Boot服务的跨域问题，使用Axios 0.27.2封装了请求/响应拦截器：请求拦截器自动注入JWT令牌，响应拦截器统一处理错误信息。

## 2.8 本章小结

本章系统介绍了萌宠乐园在线管理平台所采用的关键技术。后端以Spring Boot 2.7构建RESTful API服务，使用MyBatis-Plus 3.5.3简化数据库操作，JWT实现无状态认证，数据存储采用MySQL。前端以Vue.js 2.6构建SPA，Vue Router进行路由管理，Vuex进行状态管理，Element UI 2.15提供UI组件。上述技术的合理组合为系统设计与实现奠定了坚实基础。
